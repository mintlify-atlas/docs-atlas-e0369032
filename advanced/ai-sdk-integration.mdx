The Lexa JS SDK is built on top of the [AI SDK v2](https://sdk.vercel.ai/docs) by Vercel, providing seamless integration with the AI SDK ecosystem while giving you access to Lexa's powerful language models.

## Provider Architecture

The Lexa SDK implements the `ProviderV2` interface from AI SDK, which means you can use it with all AI SDK utilities and patterns you're familiar with.

### LexaProvider Class

The `LexaProvider` class is the main entry point for integrating Lexa models with AI SDK:

```typescript
import { LexaProvider } from '@robilabs/lexa';

const provider = new LexaProvider(
  {
    apiKey: process.env.LEXA_API_KEY,
    provider: 'lexa',
    baseURL: 'https://www.lexa.chat/api', // optional
  },
  {
    temperature: 0.7, // optional settings
    maxOutputTokens: 1000,
  }
);
```

### createLexaProvider Factory

For easier instantiation, use the factory function:

```typescript
import { createLexaProvider } from '@robilabs/lexa';

const provider = createLexaProvider(
  {
    apiKey: process.env.LEXA_API_KEY,
    provider: 'lexa',
  },
  {
    temperature: 0.7,
  }
);
```

## Getting Language Models

Once you have a provider instance, you can access language models using the `languageModel()` method:

```typescript
const model = provider.languageModel('lexa-mml');
```

## Using with AI SDK Methods

The `LexaLanguageModel` class implements `LanguageModelV2`, providing both generation and streaming capabilities:

### Text Generation

```typescript
const model = provider.languageModel('lexa-mml');

const result = await model.doGenerate({
  prompt: [
    {
      role: 'user',
      content: [{ type: 'text', text: 'Hello, how are you?' }],
    },
  ],
  temperature: 0.7,
  maxOutputTokens: 1000,
});

console.log(result.content); // Array of content parts
console.log(result.usage); // Token usage statistics
console.log(result.finishReason); // Why generation stopped
```

### Streaming

```typescript
const model = provider.languageModel('lexa-rho');

const { stream } = await model.doStream({
  prompt: [
    {
      role: 'user',
      content: [{ type: 'text', text: 'Write a short story.' }],
    },
  ],
  temperature: 0.8,
});

for await (const chunk of stream) {
  if (chunk.type === 'text-delta') {
    process.stdout.write(chunk.delta);
  } else if (chunk.type === 'finish') {
    console.log('\nUsage:', chunk.usage);
  }
}
```

## Tool Calling

Lexa models support tool calling through the AI SDK interface:

```typescript
const model = provider.languageModel('lexa-mml');

const result = await model.doGenerate({
  prompt: [
    {
      role: 'user',
      content: [{ type: 'text', text: 'What is the weather in San Francisco?' }],
    },
  ],
  tools: [
    {
      name: 'get_weather',
      description: 'Get the current weather for a location',
      parameters: {
        type: 'object',
        properties: {
          location: {
            type: 'string',
            description: 'The city and state, e.g. San Francisco, CA',
          },
        },
        required: ['location'],
      },
    },
  ],
  toolChoice: 'auto',
});

// Check for tool calls in the response
for (const content of result.content) {
  if (content.type === 'tool-call') {
    console.log('Tool:', content.toolName);
    console.log('Arguments:', content.input);
  }
}
```

## Multimodal Support

The `lexa-mml` model supports image inputs:

```typescript
const model = provider.languageModel('lexa-mml');

const result = await model.doGenerate({
  prompt: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'What is in this image?' },
        { 
          type: 'file', 
          data: 'data:image/jpeg;base64,...', // Base64 encoded image
          mimeType: 'image/jpeg' 
        },
      ],
    },
  ],
});
```

## Provider Metadata

The language model exposes metadata about the provider:

```typescript
const model = provider.languageModel('lexa-mml');

console.log(model.providerMetadata);
// {
//   lexa: {
//     model: 'lexa-mml',
//     baseURL: 'https://www.lexa.chat/api',
//   }
// }
```

## Not Yet Supported

<Warning>
The following AI SDK features are not yet supported by LexaProvider:
- Embedding models (`textEmbeddingModel()`)
- Image generation models (`imageModel()`)
</Warning>

Attempting to use these methods will throw an error:

```typescript
// This will throw an error
const embeddings = provider.textEmbeddingModel('some-model');
// Error: Text embedding models are not yet supported by LexaProvider
```
