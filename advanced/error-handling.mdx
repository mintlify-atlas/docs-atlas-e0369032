The Lexa SDK uses error types from `@ai-sdk/provider-utils` to provide consistent error handling across different failure scenarios.

## Error Types

The SDK throws two main error types, both imported from `@ai-sdk/provider-utils`:

### APICallError

Thrown for general API errors including authentication failures, invalid requests, and server errors.

```typescript
import { APICallError } from '@ai-sdk/provider-utils';

try {
  const model = provider.languageModel('lexa-mml');
  const result = await model.doGenerate({
    prompt: [/* ... */],
  });
} catch (error) {
  if (error instanceof APICallError) {
    console.error('API Error:', error.message);
    console.error('Status Code:', error.statusCode);
    console.error('Response Headers:', error.responseHeaders);
    console.error('Response Body:', error.responseBody);
  }
}
```

**APICallError Properties:**
- `message`: Human-readable error description
- `statusCode`: HTTP status code (if available)
- `responseHeaders`: HTTP response headers as an object
- `responseBody`: Raw response body
- `cause`: Original error that caused this error

### TooManyRequestsError

Thrown specifically when you hit rate limits (HTTP 429).

```typescript
import { TooManyRequestsError } from '@ai-sdk/provider-utils';

try {
  const model = provider.languageModel('lexa-mml');
  const result = await model.doGenerate({
    prompt: [/* ... */],
  });
} catch (error) {
  if (error instanceof TooManyRequestsError) {
    console.error('Rate limited!');
    console.error('Retry after:', error.retryAfter, 'seconds');
  }
}
```

**TooManyRequestsError Properties:**
- `message`: Human-readable error description
- `retryAfter`: Number of seconds to wait before retrying (if provided by API)

## Error Handling in the SDK

The SDK's error handling logic is implemented in the `LexaLanguageModel` class:

```typescript
private handleError(error: any): never {
  // Handle Response objects
  if (error instanceof Response) {
    if (error.status === 429) {
      const retryAfter = error.headers.get('retry-after');
      throw new TooManyRequestsError({
        message: 'Rate limited by Lexa API',
        retryAfter: retryAfter ? parseInt(retryAfter, 10) : undefined,
      });
    }

    throw new APICallError({
      message: `Lexa API error: ${error.status} ${error.statusText}`,
      statusCode: error.status,
      responseHeaders: Object.fromEntries(error.headers.entries()),
      responseBody: error.body,
      cause: error,
    });
  }

  // Re-throw if already an API error
  if (error instanceof APICallError || error instanceof TooManyRequestsError) {
    throw error;
  }

  // Wrap unexpected errors
  throw new APICallError({
    message: `Unexpected error: ${error.message}`,
    cause: error,
  });
}
```

## Common Error Scenarios

### Authentication Errors (401)

```typescript
import { APICallError } from '@ai-sdk/provider-utils';
import { createLexaProvider } from '@robilabs/lexa';

try {
  const provider = createLexaProvider({
    apiKey: 'invalid-key',
    provider: 'lexa',
  });
  
  const model = provider.languageModel('lexa-mml');
  await model.doGenerate({
    prompt: [{ role: 'user', content: [{ type: 'text', text: 'Hello' }] }],
  });
} catch (error) {
  if (error instanceof APICallError && error.statusCode === 401) {
    console.error('Invalid API key. Please check your credentials.');
  }
}
```

### Rate Limiting (429)

```typescript
import { TooManyRequestsError } from '@ai-sdk/provider-utils';

try {
  const model = provider.languageModel('lexa-mml');
  await model.doGenerate({
    prompt: [/* ... */],
  });
} catch (error) {
  if (error instanceof TooManyRequestsError) {
    console.log(`Rate limited. Retry after ${error.retryAfter} seconds.`);
    
    // Wait and retry
    if (error.retryAfter) {
      await new Promise(resolve => setTimeout(resolve, error.retryAfter * 1000));
      // Retry the request
    }
  }
}
```

### Server Errors (5xx)

```typescript
import { APICallError } from '@ai-sdk/provider-utils';

try {
  const model = provider.languageModel('lexa-mml');
  await model.doGenerate({
    prompt: [/* ... */],
  });
} catch (error) {
  if (error instanceof APICallError && error.statusCode >= 500) {
    console.error('Server error. The Lexa API may be experiencing issues.');
    console.error('Status:', error.statusCode);
    console.error('Message:', error.message);
  }
}
```

### Network Errors

```typescript
import { APICallError } from '@ai-sdk/provider-utils';

try {
  const model = provider.languageModel('lexa-mml');
  await model.doGenerate({
    prompt: [/* ... */],
  });
} catch (error) {
  if (error instanceof APICallError && !error.statusCode) {
    console.error('Network error. Check your internet connection.');
    console.error('Original error:', error.cause);
  }
}
```

## Implementing Retry Logic

Here's a robust retry implementation with exponential backoff:

```typescript
import { APICallError, TooManyRequestsError } from '@ai-sdk/provider-utils';

async function generateWithRetry(
  model: any,
  options: any,
  maxRetries = 3,
  baseDelay = 1000
) {
  let lastError: Error;
  
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      return await model.doGenerate(options);
    } catch (error) {
      lastError = error as Error;
      
      // Don't retry on client errors (4xx except 429)
      if (
        error instanceof APICallError &&
        error.statusCode &&
        error.statusCode >= 400 &&
        error.statusCode < 500 &&
        error.statusCode !== 429
      ) {
        throw error;
      }
      
      // Handle rate limiting
      if (error instanceof TooManyRequestsError) {
        const delay = error.retryAfter
          ? error.retryAfter * 1000
          : baseDelay * Math.pow(2, attempt);
        
        console.log(`Rate limited. Waiting ${delay}ms before retry...`);
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }
      
      // Retry on server errors (5xx) or network errors
      if (
        error instanceof APICallError &&
        (!error.statusCode || error.statusCode >= 500)
      ) {
        const delay = baseDelay * Math.pow(2, attempt);
        console.log(`Error occurred. Retrying in ${delay}ms... (Attempt ${attempt + 1}/${maxRetries})`);
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }
      
      // Don't retry on other errors
      throw error;
    }
  }
  
  throw lastError!;
}

// Usage
const model = provider.languageModel('lexa-mml');
const result = await generateWithRetry(model, {
  prompt: [
    {
      role: 'user',
      content: [{ type: 'text', text: 'Hello!' }],
    },
  ],
});
```

## Handling Streaming Errors

Errors during streaming need special handling:

```typescript
import { APICallError } from '@ai-sdk/provider-utils';

try {
  const model = provider.languageModel('lexa-mml');
  const { stream } = await model.doStream({
    prompt: [/* ... */],
  });
  
  const reader = stream.getReader();
  
  try {
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      
      if (value.type === 'text-delta') {
        process.stdout.write(value.delta);
      } else if (value.type === 'error') {
        console.error('Stream error:', value.error);
        break;
      }
    }
  } finally {
    reader.releaseLock();
  }
} catch (error) {
  if (error instanceof APICallError) {
    console.error('Failed to establish stream:', error.message);
  } else {
    console.error('Unexpected error:', error);
  }
}
```

## Abort Signal Support

The SDK supports request cancellation via `AbortSignal`:

```typescript
const controller = new AbortController();

// Cancel after 10 seconds
const timeout = setTimeout(() => controller.abort(), 10000);

try {
  const model = provider.languageModel('lexa-mml');
  const result = await model.doGenerate({
    prompt: [/* ... */],
    abortSignal: controller.signal,
  });
  
  clearTimeout(timeout);
} catch (error) {
  if (error.name === 'AbortError') {
    console.error('Request was aborted');
  } else {
    console.error('Error:', error);
  }
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always handle rate limiting">
    Implement exponential backoff and respect `retry-after` headers to avoid being blocked.
  </Accordion>
  
  <Accordion title="Log errors with context">
    Include request details, timestamps, and user context to help debug issues.
  </Accordion>
  
  <Accordion title="Don't retry client errors">
    4xx errors (except 429) indicate problems with your request. Fix the request instead of retrying.
  </Accordion>
  
  <Accordion title="Set appropriate timeouts">
    Use `AbortSignal` to prevent requests from hanging indefinitely.
  </Accordion>

  </Accordion>
</AccordionGroup>

## Error Monitoring

Integrate with error tracking services:

```typescript
import { APICallError, TooManyRequestsError } from '@ai-sdk/provider-utils';
import * as Sentry from '@sentry/node';

try {
  const model = provider.languageModel('lexa-mml');
  await model.doGenerate({
    prompt: [/* ... */],
  });
} catch (error) {
  // Log to monitoring service
  if (error instanceof APICallError || error instanceof TooManyRequestsError) {
    Sentry.captureException(error, {
      tags: {
        error_type: error.constructor.name,
        status_code: error instanceof APICallError ? error.statusCode : 429,
      },
      extra: {
        model: 'lexa-mml',
        responseHeaders: error instanceof APICallError ? error.responseHeaders : undefined,
        retryAfter: error instanceof TooManyRequestsError ? error.retryAfter : undefined,
      },
    });
  }
  
  throw error;
}
```
